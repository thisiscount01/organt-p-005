딥다이브 포트폴리오 — 시각 언어

모노스페이스를 주인공으로 한 터미널 다크 시스템. 한 줄의 코드가 초점을 잡으며 surface → bytecode → interpreter → memory 로 확장된다. 한 인물은 얼굴이 아니라 '사고의 유물'(커밋·발표·풀이·메모리)로 표현한다. 이 문서의 모든 색은 아래 라이브 대비 검사기가 실제 토큰값으로 계산해 AA 통과를 증명한다.

01 — FIRST SCREEN GATE첫 화면: 핵심 블록 ≤3 · 동급 강조 0 · 5초 판독

채용담당자가 열자마자 보는 화면. 카운트되는 블록은 딱 3개 — ①한 줄(focal) ②정체 한 문장(support) ③정체 배지(support). ambient(커서·격자)는 카운트 제외. 시선 경로는 한 줄로 흐른다.

02 — CONCEPT AXIS은유의 축: 수직낙하 아님 — '초점이 잡히며 확장'

왜 줌/해상도인가

'깊이 파고드는 사람'을 수직 스크롤로만 은유하면 흔한 관성이 된다. 대신 한 줄이 초점을 잡으며(blur→선명) 바이트코드·인터프리터·메모리 구조로 확장된다 — 해상도가 올라가는 감각이 '표면 아래를 끝까지 본다'는 정체성과 맞는다.

4깊이 = CPython 단면

surface 우리가 쓰는 한 줄 · bytecode 컴파일된 명령(PyCon 연사) · interpreter 실행 루프와 그가 바꾼 것(defer/CPython) · memory 객체·참조의 자리(백준·DICOM). 감정선: 궁금증 → 몰입 → 경외.

06 — EMPHASIS GRADESfocal · support · ambient (시각 위계 = 토큰)

focal

뷰당 1개. 시선이 착지하는 단 하나. 유일하게 액센트 허용. 카운트 O.

support

뷰당 ≤2. focal을 뒷받침. 액센트 채움 금지 — 텍스트 위계로만. 카운트 O.

ambient

배경 텍스처(흐르는 opcode·격자). 카운트 X, aria-hidden. 시선 안 뺏음.

→ 첫 화면 카운트 블록 = focal(1) + support(≤2) ≤ 3. 동급 강조 0 보장.

09 — PERSON AS ARTIFACT한 인물 = 사고의 유물 (표현 규칙)

DO — 사고의 궤적으로

✓얼굴 사진 대신 커밋·발표 슬라이드·바이트코드·풀이 흔적·메모리 셀을 유물처럼.
✓모든 유물은 실제 사실 하나를 담는다(PyCon·CPython·백준·DICOM). 과장 0.
✓유물은 모노스페이스 + 터미널/코드 프레임. 액센트는 '살아있는' 한 점에만.

DON'T

✕인물 사진·아바타·스톡 일러스트로 인물을 '설명'하지 않는다.
✕기능 6칸 그리드·정적 스크린샷 나열(동급 강조 남발) 금지.
✕유물 없는 장식(순수 파티클·의미 없는 3D) 금지 — 모든 시각은 실체를 가리킨다.
